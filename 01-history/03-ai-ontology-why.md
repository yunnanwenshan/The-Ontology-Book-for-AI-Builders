# 1.3 · AI 时代为什么需要本体（必读）

> **一句话**：LLM 是个会编故事的算命先生。本体是给算命先生的"真经"。

## LLM 的 3 个病（本体都能治）

| 病 | 症状 | 本体怎么治 |
|---|------|-----------|
| **幻觉** | 一本正经说错 | LLM 不知道时去本体查，查不到就说"不知道" |
| **数据孤岛** | 销售数据在 MySQL，物流在 PG，订单在 Oracle | 一张图统一 schema，跨源查询 |
| **黑盒** | 答对了但不知道为什么 | 每条结论附本体路径，审计可回放 |

## 5 分钟判断：你的场景要不要本体

| 场景 | 要本体？ | 为什么 |
|------|---------|--------|
| 通用聊天 | ❌ | 没结构化数据 |
| 客服（订单查询） | ✅✅ | 订单结构化、必须准 |
| 文档问答 | ⚠️ | 文档为主，本体辅助 |
| 决策支持 | ✅✅✅ | 不允许错 |
| 跨组织数据 | ✅✅✅ | 双方要共同 schema |
| 多 Agent 协作 | ✅✅ | Agent 共享世界模型 |

**口诀**：
- 数据是结构化的 → 强需要
- 结果不允许错 → 强需要
- 数据跨系统 → 强需要
- 都不需要 → RAG 更省事

## 决策树

```
你的数据有结构化 schema 吗？
├── 否 → 用 RAG
└── 是
    ├── 准确率 95%+？→ 本体 + SPARQL
    └── 灵活为主？ → RAG / GraphRAG
```
