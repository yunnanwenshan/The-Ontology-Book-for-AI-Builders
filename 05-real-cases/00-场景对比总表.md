# 8 个实战案例怎么选

> **从易到难**：HR < 客服 < 商品 < 财务 < 物流 < CRM < 医疗 < Foundry 平台
> 不知道选哪个？**从第一个开始，逐个跑**

## 选案例的 3 步法

```
1. 你的问题能在"业务问题"表里找到？→ 直接用
2. 你的行业能在"行业"表里找到？→ 用对应组合
3. 还是不确定？→ 从 HR 开始（最简单）
```

## 跑一个案例的标准流程

```bash
# 1. 进目录
cd 05-real-cases/hr/    # 换成你选的案例

# 2. 读 README 了解业务问题
cat README.md

# 3. 跑主脚本
python3 hr-query.py    # 换成 case-query.py / run_platform.py 等

# 4. 看输出
# 应该看到 "数据校验：✅ 通过" + 6 个查询结果
```

**跑通了 = 学会了一个完整场景**。再换下一个。

## 共同模式

每个案例都有这 4 个文件：

```
case-name/
├── README.md            ← 业务问题 + 跑法
├── case-query.py        ← 跑通的查询
└── data/
    ├── case-ont.ttl     ← 本体（核心类 + 关系）
    ├── case-shapes.ttl  ← SHACL 业务规则
    └── case-data.ttl    ← 业务数据
```

**例外：Foundry 平台** 多了 6 个脚本（核心 / AIP / 物化 / 仪表盘 / ABAC / CDC），对应 Palantir Foundry 的 6 大能力。

## 7 个案例一起跑

```bash
# 全部跑一遍
for d in 05-real-cases/hr 05-real-cases/customer-service 05-real-cases/finance \
         05-real-cases/product 05-real-cases/medical 05-real-cases/logistics \
         05-real-cases/crm 05-real-cases/foundry-platform; do
    echo "=== $d ==="
    cd "$d" && python3 $(ls *-query.py run_platform.py 2>/dev/null | head -1) 2>&1 | head -3
    cd - > /dev/null
done
```

## 把案例接到你公司

每个 `case-data.ttl` 都可以换：
1. 把你的真实数据导出成 Excel/CSV
2. 用 `04-zero-install-demo/examples/2-csv-to-rdf.py` 转 RDF
3. 把数据替换到 `data/case-data.ttl`
4. 跑 `case-query.py` 看你业务的 6 个查询
