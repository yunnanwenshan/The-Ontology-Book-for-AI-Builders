## 关键问题（按严重度排序）

### 🔴 P0 · 必须修（影响小白能否用）

#### 1. 顶层 README 描述和数据不同步
**问题**：README 写"5 个真实业务案例"实际有 7 个；写"4 个 Foundry 模块"实际有 6 个
**影响**：小白以为项目"小"，跑一遍发现 20 个脚本，认知失调
**修法**：顶部"已交付"卡片，**数字 + 一行话**自动反映真实状态

#### 2. 哲学章对小白的"阅读价值"低
**问题**：第 1 章 70% 内容讲 2400 年哲学史（亚里士多德 → 康德 → 弗雷格）
**影响**：小白翻两页就放弃（"这跟我做项目有什么关系？"）
**修法**：哲学章缩短到 1 屏（3 句话），**前置一个"哲学 vs 工程"小节**，让小白知道为什么看

#### 3. "02-specs" 和 "02-实战指南" 重复
**问题**：RDF 教程在 `02-specs/01-rdf.md` 讲，"手把手"在 `02-实战指南/2.1-rdf-hands-on.md` 重复讲
**影响**：小白不知道先看哪个，看完一个再看另一个觉得"重复"
**修法**：`02-specs/` 改为**速查手册**（只列语法/例子），`02-实战指南/` 改为**主线教程**（带场景带练习）

#### 4. 7 个案例没有梯度
**问题**：HR/客服/财务/商品/医疗/物流/CRM 都是"中等复杂度"，没有"极简入门"
**影响**：小白不知道先看哪个
**修法**：明确标**入门 → 进阶**：
- **入门级**（1 小时跑通）：HR、客服
- **中级**（半天深入）：财务、商品
- **高级**（需要业务背景）：医疗、物流、CRM
- **集成级**：Foundry

#### 5. Foundry 平台 README 严重滞后
**问题**：README 写 4 个模块，实际有 6 个（漏了 abac_security.py、cdc_demo.py）
**影响**：小白按 README 跑，发现还有 2 个没提到
**修法**：重新写 README，包含全部 6 个模块

### 🟡 P1 · 应该修（影响体验）

#### 6. 没有"作者手记"
**问题**：没有任何地方说"我（作者）怎么学的、踩了什么坑、推荐顺序"
**影响**：小白读完不知道怎么用
**修法**：新增 `00-overview/AUTHOR-NOTES.md`，讲清思路

#### 7. 缺"如果跑不出来"故障排查
**问题**：每个 demo 只说"应该看到 XXX"，但**如果看到的是错误**怎么办？
**影响**：小白遇到 ImportError / ModuleNotFoundError 不知道怎么继续
**修法**：在 `04-zero-install-demo/README.md` 加 "故障排查" 节，覆盖前 5 个常见错误

#### 8. PDF 教程"第 9 章"和实际案例"内容不同步"
**问题**：PDF 说 7 个案例、4 个 Foundry 模块，实际 7+8 个案例、6 个 Foundry 模块
**影响**：打印 PDF 后发现不对
**修法**：每次改 README 时，**同时跑一次 PDF 生成**

#### 9. "my-rdf.py"等"练一下"环节没说清文件放哪
**问题**：2.1 实战指南说"新建 my-rdf.py"——但**没说放哪个目录**
**影响**：小白可能放在根目录，跑的时候找不到
**修法**：明确说"放在 `04-zero-install-demo/` 下"

#### 10. 所有案例开头都是 6 个"业务问题"——但**没有说"这些问题其实没真用"
**问题**：HR 案例写"HR 总监问这 6 个问题"——但 6 个查询是开发者跑给 HR 看的
**影响**：误解"业务负责人会用这套"
**修法**：加一节"这套系统是给开发者用还是给业务用"说明

### 🟢 P2 · 可选（打磨）

#### 11. README 太长（149 行）
**问题**：项目根目录 README 一次性堆了所有信息
**修法**：缩短到 50 行，**3 条路径 + 总入口图**

#### 12. 缺"作者推荐阅读顺序"图
**问题**：小白不知道 100 个文件怎么读
**修法**：画一张"阅读路线图"（PDF 风格）

#### 13. "00-场景对比总表.md"和 README 内容重叠
**问题**：两个文件都说"7 个案例怎么选"
**修法**：README 删掉对比表，只链接到 00-场景对比总表

#### 14. 缺"修改历史"（CHANGELOG）
**问题**：不知道哪个版本有哪些内容
**修法**：加 `CHANGELOG.md`

## Review 总结

**内容已 90% 完成**，但**"小白引导"**是最大短板。

最大的"小白"障碍是：
1. **目录结构让人迷路**（10 个目录，100+ 文件）
2. **章节重复**（教程版 + 手把手版讲一样的事）
3. **没明示"按什么顺序读"**

修了 P0 三个问题后，**小白从"完全不知道看什么"变成"看 README 就知道先看 1 步"**，这是**最大的体验提升**。
